HTML

Introduction

What is HTML?

HTML stands for HyperText Markup Language. It's basically the skeleton of every website, it tells the browser what goes on the page. You use tags to wrap your content, like putting labels on boxes so the browser knows what's inside.

Explain the head and the body tags

Head tag: This is where you put stuff the browser needs to know but doesn't actually show the user. Things like the page title, what language the text is in, or links to your CSS file go here.

Body tag: Everything the user actually sees goes in here: your text, images, buttons, all of it. If it shows up on screen, it's in the body.

Basic Tags

Semantic Tags

  • section: Used to group related stuff together on the page
  • article: For content that makes sense on its own, like a blog post or a news story
  • aside: Side content, like a sidebar or a fun fact that doesn't fit in the main flow
  • header: Usually has the logo, navigation, or intro stuff at the top
  • footer: The bottom of the page, copyright, links, contact info, that kind of thing

CSS

Introduction

What is CSS?

CSS stands for Cascading Style Sheets. While HTML gives the page its structure, CSS is what makes it look good (or in this case, really bad). You use it to change colors, fonts, sizes, spacing, basically anything visual.

What is the purpose of CSS?

The main idea is to keep your styling separate from your content. That way you can change how the whole site looks without touching the HTML. It also makes it way easier to keep things consistent across multiple pages.

How to apply CSS to HTML (3 options)

  1. Inline CSS: You write the style right inside the tag. Like <p style="color: blue;">. It works but gets messy fast.
  2. Internal CSS: You write all your styles in a <style> tag inside the <head>. Better than inline, but still only applies to one page.
  3. External CSS: You make a separate .css file and link it with <link rel="stylesheet" href="styles.css">. This is the right way to do it.

What is a selector and CSS rule?

Selector: It's how you tell CSS which element to style. Could be a tag like p, a class like .box, or an id like #header.

CSS Rule: It's the selector plus the actual styles. Looks like this: selector { property: value; }. Pretty simple once you get the hang of it.

Basic CSS

Selectors

  • Tag Selector: Targets every element of that type. Like p { color: blue; } makes all paragraphs blue.
  • Class Selector: Targets elements with a specific class. .highlight { background: yellow; } only affects things with class="highlight".
  • Tag and Class Selector: You can combine them. p.highlight { color: red; } only targets paragraphs with that class, not other elements.
  • ID Selector: Targets one specific element. #header { font-size: 24px; }. IDs should only be used once per page.

Class vs ID

Class: You can reuse it on as many elements as you want. Great for styling groups of things the same way.

ID: Should only be on one element per page. Use it when something is unique, like the main nav or a specific section you need to target.

CSS Box Model

Every element on the page is basically a box. From the inside out it goes: content, then padding, then border, then margin. Understanding this makes it way easier to figure out why things don't line up the way you expect.

Media Queries

Media queries let you apply different styles depending on the screen size. So you can make things look decent on both a phone and a desktop. Example: @media (max-width: 768px) { body { font-size: 14px; } }

Layout CSS

Display: Grid

Grid lets you build layouts with rows and columns at the same time. It's great for bigger page layouts. You define how many columns you want and the browser figures out the rest.

Display: Flex

Flexbox is for arranging things in a single row or column. Super useful for navbars, card rows, or centering stuff. Once it clicks it's hard to go back to not using it.
